# 데이터 모델 명세서

## 1. 개요

이 데이터 모델은 수강 신청 시스템의 핵심 데이터인 회원, 강의, 수강 신청을 분리하여 관리하기 위한 구조입니다.

| 테이블           | 목적               |
|---------------|------------------|
| `members`     | 서비스를 사용하는 회원 정보  |
| `courses`     | 크리에이터가 개설한 강의 정보 |
| `enrollments` | 회원의 수강 신청 기록     |

---

## 2. ERD

```mermaid
erDiagram
    members ||--o{ courses : creates
    members ||--o{ enrollments : applies
    courses ||--o{ enrollments : has

    members {
        BIGINT id PK
        VARCHAR email UK
        VARCHAR password_hash
        VARCHAR name
        VARCHAR role
        VARCHAR status
        DATETIME created_at
        DATETIME updated_at
    }

    courses {
        BIGINT id PK
        BIGINT creator_id
        VARCHAR title
        TEXT description
        DECIMAL price
        INT capacity
        INT seat_left_count
        DATETIME period_start
        DATETIME period_end
        VARCHAR status
        DATETIME created_at
        DATETIME updated_at
    }

    enrollments {
        BIGINT id PK
        BIGINT course_id
        BIGINT member_id
        VARCHAR status
        DATETIME payment_pending_start_at
        DATETIME payment_pending_end_at
        DATETIME created_at
        DATETIME updated_at
    }
````

## 3. 테이블 명세

### 3.1. members

`members`는 서비스 사용자를 저장하는 테이블입니다.

회원의 로그인 정보, 이름, 역할, 상태를 관리합니다.

| 컬럼              | 목적                |
|-----------------|-------------------|
| `id`            | 회원 식별자입니다.        |
| `email`         | 로그인에 사용하는 이메일입니다. |
| `password_hash` | 비밀번호 해시 값입니다.     |
| `name`          | 회원 이름입니다.         |
| `role`          | 회원 역할입니다.         |
| `status`        | 회원 상태입니다.         |
| `created_at`    | 생성 시각입니다.         |
| `updated_at`    | 수정 시각입니다.         |

주요 제약조건은 다음과 같습니다.

| 제약조건               | 목적                |
|--------------------|-------------------|
| `PRIMARY KEY (id)` | 회원을 고유하게 식별합니다.   |
| `UNIQUE (email)`   | 이메일 중복 가입을 방지합니다. |

---

### 3.2. courses

`courses`는 강의 정보를 저장하는 테이블입니다.

강의의 생성자, 제목, 설명, 가격, 정원, 수강 기간, 모집 상태를 관리합니다.

| 컬럼                | 목적                  |
|-------------------|---------------------|
| `id`              | 강의 식별자입니다.          |
| `creator_id`      | 강의를 생성한 회원의 식별자입니다. |
| `title`           | 강의 제목입니다.           |
| `description`     | 강의 설명입니다.           |
| `price`           | 강의 가격입니다.           |
| `capacity`        | 최대 수강 가능 인원입니다.     |
| `seat_left_count` | 현재 남은 좌석 수입니다.      |
| `period_start`    | 수강 기간 시작 시각입니다.     |
| `period_end`      | 수강 기간 종료 시각입니다.     |
| `status`          | 강의 모집 상태입니다.        |
| `created_at`      | 생성 시각입니다.           |
| `updated_at`      | 수정 시각입니다.           |

주요 인덱스는 다음과 같습니다.

| 인덱스                  | 목적                       |
|----------------------|--------------------------|
| `INDEX (creator_id)` | 특정 크리에이터가 생성한 강의를 조회합니다. |
| `INDEX (status)`     | 강의 목록을 모집 상태 기준으로 조회합니다. |

---

### 3.3. enrollments

`enrollments`는 회원의 수강 신청 기록을 저장하는 테이블입니다.

회원과 강의를 직접 연결하지 않고, 수강 신청을 독립적인 데이터로 관리합니다.

| 컬럼                         | 목적                |
|----------------------------|-------------------|
| `id`                       | 수강 신청 식별자입니다.     |
| `course_id`                | 신청 대상 강의의 식별자입니다. |
| `member_id`                | 신청한 회원의 식별자입니다.   |
| `status`                   | 수강 신청 상태입니다.      |
| `payment_pending_start_at` | 결제 대기 시작 시각입니다.   |
| `payment_pending_end_at`   | 결제 대기 만료 시각입니다.   |
| `created_at`               | 생성 시각입니다.         |
| `updated_at`               | 수정 시각입니다.         |

주요 제약조건과 인덱스는 다음과 같습니다.

| 제약조건 / 인덱스                      | 목적                         |
|---------------------------------|----------------------------|
| `PRIMARY KEY (id)`              | 수강 신청을 고유하게 식별합니다.         |
| `UNIQUE (course_id, member_id)` | 같은 회원의 같은 강의 중복 신청을 방지합니다. |
| `INDEX (member_id)`             | 회원별 수강 신청 목록을 조회합니다.       |
| `INDEX (course_id)`             | 강의별 수강 신청 목록을 조회합니다.       |
| `INDEX (course_id, status)`     | 강의별 신청 데이터를 상태 기준으로 조회합니다. |

결제 대기 만료 처리를 효율적으로 조회하려면 다음 인덱스를 추가할 수 있습니다.

| 인덱스                                      | 목적                          |
| ---------------------------------------- | --------------------------- |
| `INDEX (status, payment_pending_end_at)` | 만료 대상인 `PENDING` 신청을 조회합니다. |

---

## 4. 관계 명세

| 관계                          | 기준 컬럼                   | 목적                                 |
|-----------------------------|-------------------------|------------------------------------|
| `members` 1:N `courses`     | `courses.creator_id`    | 한 회원이 여러 강의를 생성할 수 있음을 표현합니다.      |
| `members` 1:N `enrollments` | `enrollments.member_id` | 한 회원이 여러 강의에 신청할 수 있음을 표현합니다.      |
| `courses` 1:N `enrollments` | `enrollments.course_id` | 하나의 강의에 여러 신청 기록이 연결될 수 있음을 표현합니다. |

---

## 5. 설계 요약

| 항목        | 설명                                                                                    |
|-----------|---------------------------------------------------------------------------------------|
| 회원 데이터    | `members`에서 관리합니다.                                                                    |
| 강의 데이터    | `courses`에서 관리합니다.                                                                    |
| 수강 신청 데이터 | `enrollments`에서 관리합니다.                                                                |
| 강의 생성자    | `courses.creator_id`로 식별합니다.                                                          |
| 수강 신청 관계  | `enrollments.course_id`, `enrollments.member_id`로 표현합니다.                              |
| 정원 데이터    | `courses.capacity`, `courses.seat_left_count`로 관리합니다.                                 |
| 중복 신청 제한  | `UNIQUE (course_id, member_id)`로 처리합니다.                                               |
| 결제 대기 시간  | `enrollments.payment_pending_start_at`, `enrollments.payment_pending_end_at`으로 관리합니다. |

이 구조는 회원, 강의, 수강 신청을 각각 독립적인 데이터로 분리하여 관리하기 위한 모델입니다.
